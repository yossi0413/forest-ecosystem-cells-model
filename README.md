# 🌲 森林生態系シミュレーター 超・完全技術仕様書

本ドキュメントは、森林遷移および土壌動態シミュレーターの内部アルゴリズム、数値計算、および動作規則を網羅した公式仕様書です。

---

## 1. システム概要
本システムは、**植物のライフサイクル**、**土壌の栄養循環**、および**突発的な環境攪乱（災害）**の3要素が相互に影響し合うセル・オートマトン・モデルです。

*   **グリッドサイズ**: 100 × 100（合計 10,000マス）
*   **時間単位**: 1ステップ ＝ 1年
*   **レイヤー構造**:
    1.  **植生レイヤー**: 植物の状態（陽樹、陰樹、枯れ木など）を保持
    2.  **土壌レイヤー**: 各マスの栄養度（0.0 〜 500.0以上）を保持
*   **描画**: HTML5 Canvas（`image-rendering: pixelated` により鮮明なセル表示を実現）

## 3. 数値計算アルゴリズム

### 3.1 植物の遷移ロジック
毎ステップ、各マスで以下の判定が優先順に行われます。
1.  **芽吹き（EMPTY時）**:
    *   **一次遷移**: 栄養度 < `primarySoilThreshold` の場合、`primarySuccessionProb` で陽樹が発生。
    *   **二次遷移**: 栄養度 >= `secondarySoilThreshold` の場合、陽樹発生率が `sunSeedProb * richSoilBoost` に向上。さらに `shadeSeedProb` で陰樹も発生可能。
    *   **通常時**: `sunSeedProb` または `shadeSeedProb` で発生。
2.  **陽樹の淘汰と交代**:
    *   **被陰ストレス**: 隣接8マスに樹木が `shadeThreshold` 個以上ある場合、陽樹は `shadeStressProb` の確率で枯死。
    *   **種の交代**: 陽樹のマスに陰樹の種が飛来（`shadeSeedProb`）すると、陰樹に置き換わる。
    *   **自然寿命**: `sunLifeProb` で枯死。
3.  **陰樹の安定**:
    *   **自然寿命**: `shadeLifeProb` という極めて低い確率でのみ枯死。一度成立すると非常に安定。

### 3.2 土壌の代謝（Soil Metabolism）
土壌栄養度 $S$ の更新式：
$$S_{next} = \text{clamp}(S_{current} + \text{Diff} - \text{Cons} + \text{Gain}, 0, \text{maxSoilFertility})$$
*   **拡散 (Diffusion)**: 隣接4マスの平均との差の `soilDiffusionRate` が混合。
*   **消費 (Consumption)**: 樹木が存在する場合、毎年 `-treeSoilConsumption` 減少（陽樹は1.5倍消費）。
*   **還元 (Gain)**:
    *   枯れ木の分解: `+deadTreeSoilGain`（分解確率 `deadTreeDecompProb`）。
    *   灰の定着: `+ashSoilGain`（火災・火山後）。
    *   岩の風化: `+rockWeatheringRate`（不毛な地からの回復）。

## 5. 攪乱（災害）システム

### 5.1 山火事と延焼（Wildfire）
延焼確率 $P_{spread}$ は、周囲の可燃物（樹木）密度によって変動します。
$$P_{spread} = fireSpreadProb \times (0.2 + 0.8 \times \frac{\text{周囲の樹木数}}{8})$$
※陰樹の森は樹冠が密接しているため、さらに `1.3倍` の燃えやすさ補正がかかります。

### 5.2 台風（Typhoon）
ランダムな位置から発生し、150サイクルにわたってブラウン運動的に移動します。中心半径内の樹木を強制的に「枯れ木」に変換し、森林にギャップ（空き地）を作ります。

### 5.3 火山噴火（Volcano）
ランダムな地点を中心に円形に「溶岩」を噴出し、中心部の土壌栄養度を強制的に `0` にリセットします。地表は一時的に「岩」になり、長い年月をかけて風化を待つことになります。

## 6. 統計指標と可視化

### 6.1 植生多様性 (Vegetation Diversity)
陽樹（SUN_TREE）と陰樹（SHADE_TREE）の比率のみに着目した指標。
$$D_{veg} = -\frac{p_{sun} \ln p_{sun} + p_{shade} \ln p_{shade}}{\ln 2}$$
正規化されたシャノン指数であり、陽樹と陰樹が 1:1 で混在している状態で最大値 1.0 を取ります。

### 6.2 空間モザイク指数 (Spatial Mosaic Index)
隣接するセル同士の状態が異なる「境界線」的密度を評価します。
$$Mosaic = \frac{\text{状態が異なる隣接境界数}}{\text{全境界数}} \times \text{補正係数}$$
単一の種が広がる均一な森よりも、パッチワーク状に多様な状態が入り混じっている複雑な景観を高く評価します。

### 6.3 土壌の不均一性指数 (Soil Heterogeneity)
全セルの土壌栄養度の標準偏差を指標化したものです。
$$Heterogeneity = \frac{\sigma_{soil}}{50} \text{ (max 1.0)}$$
分解や消費のムラによって栄養度が高い場所と低い場所が混在している（環境のポテンシャルが多様である）状態を示します。平均的な森林では 0.3〜0.5 前後を推移します。

### 6.4 ギャップ動態率 (Gap Dynamics Rate)
樹木の枯死（寿命、ストレス、災害）によって新たに発生したギャップの割合。
$$GapRate = \frac{\text{その年に発生した枯死数}}{\text{全マス数} \times 0.05}$$
森が停滞せず、常に新陳代謝（世代交代）を繰り返している活発さを評価します。

### 6.5 過酷度 (Severity)
環境のストレス度。溶岩、火災、岩、灰などの被災面積を重み付けして算出。
$$Severity = \min(1.0, \sqrt{\sum (\text{状態の割合} \times \text{重み})} \times 1.5)$$
※重み: 溶岩(1.0), 火災(0.8), 岩(0.6), 灰(0.3), 枯れ木(0.1)。森林全体（実線）と局所エリア（紫の点線）の両方で計算されます。

### 6.6 局所観測エリア (Monitoring Area)
グリッド上の赤い枠（10x10マス）は、特定の地点を集中的に観測するためのエリアです。
*   **位置**: シミュレーション開始またはリセット時にランダムに決定されます。
*   **グラフ表示**: このエリア内の指標は、対応する色の**点線**で表示されます。
### 6.7 総合評価システム (Ecosystem Evaluation System)
パラメータ変更後100年間の多様性指標をサンプリングし、その平均値を「生態系スコア」として算出・記録します。

*   **サンプリング期間**: 変更後 1〜100年（この間はUIに「調査中」と表示）。
*   **算出式**: 
    $$Score_{avg} = \frac{\sum_{t=1}^{100} Diversity_t}{100}$$
*   **評価履歴**: 100年経過後の結果は「評価履歴チャート」に永続化され、異なる設定間での生物多様性維持能力を視覚的に比較できます。

### 6.8 土壌ビュー（Heatmap Color）
栄養度 $V$ (0-500) を以下の6段階で補完し、高濃度な栄養状態まで可視化します。
1.  **0-25**: 青 → 水色 (貧栄養)
2.  **25-50**: 水色 → 緑
3.  **50-75**: 緑 → 黄
4.  **75-100**: 黄 → 赤 (肥沃)
5.  **100-200**: 赤 → **紫** (超肥沃)
6.  **200-500**: 紫 → **黒** (究極の肥沃・腐植層)

## 7. 技術的実装
*   **描画の最適化**: 1万セルの `image-rendering: pixelated` Canvas描画。
*   **タイムトラベル**: 10年ごとに `grid` と `soilGrid` をディープコピーして `historySnapshots` に格納。
*   **チャート**: `Chart.js` を使用し、直近履歴をリアルタイム更新。
*   **シミュレーション速度**: `requestAnimationFrame` 内のタイムスタンプ比較により、10ms〜1000msの可変速を実現。

## 8. プリセット設定 (Climate Presets) 解説

### 8.1 熱帯雨林 (Tropical Forest)
**「圧倒的な成長力と再生スピード」**
高温多湿。樹木の成長が極めて速く、攪乱が起きてもすぐに回復します。栄養循環が激しく、常に緑が溢れる環境です。

### 8.2 乾燥草原 / 砂漠 (Desert)
**「不毛の地と過酷な生存競争」**
乾燥し、土壌栄養が極めて乏しい環境。一度火災が起きると回復に膨大な時間がかかり、わずかな資源を巡る厳しい生存戦略が求められます。

### 8.3 亜寒帯 / 高山ガレ場 (Boreal / Alpine)
**「停滞する時間と陰樹の粘り」**
寒冷で成長が遅い環境。陽樹の寿命が短く、日陰に強い陰樹が長い年月をかけてゆっくりと支配を広げていく、静かな遷移が特徴です。

### 8.4 火山帯 (Volcanic Zone)
**「破壊と驚異的な肥沃化」**
最高頻度の噴火が発生。溶岩でリセットされますが、火山灰による土壌還元が凄まじく、破壊の直後から驚異的なスピードで緑が回復する環境です。

### 8.5 サバナ (Savanna)
**「日陰なき乾燥草原」**
陽樹（低木）がまばらに広がる環境。日陰が致命的なストレスとなるため陰樹はほぼ育たず、定期的な野火によって「開けた景観」が維持されます。

### 8.6 仮想環境 (Barren Silence / Undersea)
*   **不毛の静寂 (Moon)**: ほぼすべての変化が停止した、静止した景観。最小値設定により極稀に変化が生じます。
*   **海底の森 (Underwater)**: 火災がなく、水生植物特有の急速な広がりをシミュレートします。
